#This is a web project use Python Django & React & javascript
Back-end:
1. Before use django need use command to intall it.
            command : pip3 install django djangorestframework
2. Create a project folder(music_controller) under the framwork folder. 
            command : django-admin startproject music_controller
3. Move to project folder 
            command : cd music_controller/
4. Make a app name as api inside the project folder to use to handle our api
            command : django-admin startapp api
5. Add two apps to project folder, write at in file settings.py.
            At lines(# Application definition): "INSTALLED_APPS"=[...,'api.apps.ApiConfig', 'rest_framework',] 
Now, we can try to create the 1st view at app.

# How to run the web app?
6. Before run the web server, use command to update the model and database(or init app)
            command : python3 manage.py makemigrations
7. Migrate to apply what we migrations
            command : python3 manage.py migrate
8. Finally, run the web server use the following
            command : python3 manage.py runserver
Now, try your web app what you created.

Front-end:
1. Before use react need use command to check the npm amd node.js are exsit or not.
            command : npm -v or npm
2. Create a new app inside the music_controller project as front-end.
            command : django-admin startapp frontend
3. Then into the new app.
            command : cd frontend/
4. Create some folders inside the frontend app.
            +templates
            +src    -| +components(react components)
            +static -| +frontend (store javascript bundle)
                     | +css
                     | +images
5. Init npm project, the command to create a node modules package json and other we need.
            command : npm init -y
6. Install something from npm use commands.
            webpack:      npm i webpack webpack-cli --save-dev
            babel:        npm i @babel/core @babel/cli babel-loader @babel/preset-env @babel/preset-react --save-dev
            react:        npm i react react-dom --save-dev
            async&await:  npm i @babel/plugin-proposal-class-properties
            reroute page: npm install react-router-dom
            material-ui:  npm install @mui/material @emotion/react @emotion/styled
            icon:         npm install @mui/icons-material

```Note: this v4 @material-ui/core is for react 16/17.
@mui/material 是新版 MUI（v5），架构上拆分了“组件库”与“样式引擎”；因此通常需要同时安装样式运行时包（常用的是 Emotion）。
你看到的三个包：
@mui/material —— MUI 组件库（等同于旧版的 core，但 API 有变化）
@emotion/react —— Emotion 的运行时（CSS-in-JS 运行时）
@emotion/styled —— Emotion 的 styled 工具（方便写 styled 组件）```

####View.py 相关API
APIView 中的方法名对应 HTTP 方法：

def get(self, request, code):    # 处理 GET 请求
def post(self, request, code):   # 处理 POST 请求
def put(self, request, code):    # 处理 PUT 请求
def delete(self, request, code): # 处理 DELETE 请求

generics.ListAPIView 中的方法名是 DRF 的钩子函数：
def get_queryset(self):     # DRF 调用这个方法获取数据
def get_serializer(self):   # DRF 调用这个方法获取序列化器
def get_permissions(self):  # DRF 调用这个方法获取权限


1. 首先理解 Git 的基本概念
Git 就像代码的"时光机"：

本地保存代码版本

GitHub/GitLab 是云端存储代码的地方
git push = 上传代码到云端
git pull = 从云端下载代码

Git 不跟踪的文件：
❌ db.sqlite3（数据库）
❌ media/（用户上传的图片）
❌ 敏感信息（密码、密钥）
❌ Node.js 依赖包
Git 跟踪的文件：
✅ Python 代码
✅ React 代码
✅ 配置文件
✅ 文档
